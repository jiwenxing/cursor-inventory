# 采购订单优化 - 功能说明

## 新增功能

### 1. 从销售订单创建采购订单（按供应商分组）

在销售订单列表页面，每个订单的操作栏增加了「采购」按钮，点击后可以：
- 查看该订单商品的采购建议（按供应商自动分组）
- 调整每个商品的采购数量和单价
- 选择/取消选择某些供应商或商品
- **一次性为多个供应商创建采购订单**

### 2. 采购建议（分组展示）

采购建议对话框按供应商分组展示：

**分组展示：**
- 每个供应商一个折叠面板
- 显示该供应商的商品数量和小计金额
- 可以展开/折叠查看详情

**每个商品展示以下信息：**
| 字段 | 说明 |
|------|------|
| 商品 | 商品名称和型号 |
| 销售数量 | 销售订单中的数量 |
| 当前库存 | 仓库中该商品的现有库存 |
| 建议采购量 | 销售数量 - 当前库存（如果库存充足则为 0） |
| 采购数量 | 可调整，用户实际要采购的数量 |
| 采购单价 | 可调整，默认使用商品的采购价 |
| 金额 | 采购数量 × 采购单价 |

**交互功能：**
- 可以勾选/取消勾选整个供应商（该供应商所有商品）
- 可以勾选/取消勾选单个商品
- 可以删除某个供应商分组
- 实时显示选中的供应商数量和总金额

### 3. 采购状态管理

采购订单明细增加了采购状态字段，包含三种状态：
- **待下单**：初始状态，表示采购计划已创建但尚未发给供应商
- **待确认**：已发给供应商，等待供应商确认
- **已下单**：供应商已确认，可以准备收货

**状态约束**：只有采购状态为「已下单」的商品才能进行入库操作。

### 4. 来源订单追踪

采购订单明细记录了来源销售订单 ID，在展开明细中可以查看：
- 来源订单号（SO-xxx）
- 便于追踪采购需求的来源

## 后端 API

### 新增接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/sales-orders/{order_id}/purchase-suggestions` | 获取销售订单的采购建议（**已更新：按供应商分组返回**） |
| POST | `/purchase-orders/batch-from-sales-order/{sales_order_id}` | **批量创建**采购订单（按供应商分组） |
| POST | `/purchase-orders/from-sales-order/{sales_order_id}` | 从销售订单创建采购订单（保留向后兼容） |
| PUT | `/purchase-orders/{order_id}/items/{item_id}/purchase-status` | 修改采购明细状态 |
| POST | `/purchase-orders/{order_id}/receive` | 采购入库（带约束检查） |

### 数据模型变更

**`purchase_order_items` 表新增字段**：
- `source_sales_order_id` (INTEGER, nullable) - 来源销售订单 ID
- `purchase_status` (VARCHAR(20), default '待下单') - 采购状态

## 前端变更

### SalesOrders.vue
- 操作列增加「采购」按钮
- 新增采购建议对话框组件
- 实现 `handlePurchase`、`handleCreatePurchaseOrder` 方法

### PurchaseOrders.vue
- 展开明细增加采购状态显示和编辑
- 展开明细增加来源订单号显示
- 新增入库对话框
- 实现 `handlePurchaseStatusChange`、`handleReceive`、`handleConfirmReceive` 方法
- 删除操作增加约束检查（已下单的商品不能删除）

## 约束规则

### 入库约束
- 只有采购状态为「已下单」的商品才能入库
- 尝试入库非「已下单」状态的商品时，返回错误提示：
  ```
  商品「xxx」采购状态为「待下单」，尚未下单给供应商，无法入库
  ```

### 删除约束
- 已有入库记录的订单不能删除
- 已有商品下单给供应商（采购状态为「已下单」）的订单不能删除

## 数据库迁移

执行以下命令进行数据库迁移：

```bash
cd backend
source venv/bin/activate
python migrate_purchase_source.py
```

迁移内容：
- 添加 `source_sales_order_id` 字段
- 添加 `purchase_status` 字段
- 更新现有数据（已有入库记录的商品设为「已下单」）

## 使用流程

1. **创建销售订单**
   - 在销售订单页面创建客户订单

2. **生成采购建议**
   - 点击销售订单操作栏的「采购」按钮
   - 系统根据销售数量和库存自动生成采购建议

3. **创建采购订单**
   - 选择供应商
   - 调整采购数量和单价
   - 确认创建采购订单

4. **跟踪采购状态**
   - 在采购订单列表展开明细
   - 修改采购状态：待下单 → 待确认 → 已下单

5. **采购入库**
   - 点击「入库」按钮
   - 只有「已下单」状态的商品才能入库
   - 输入入库数量，确认入库
